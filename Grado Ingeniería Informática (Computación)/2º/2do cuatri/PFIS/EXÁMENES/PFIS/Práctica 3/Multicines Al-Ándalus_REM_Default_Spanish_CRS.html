1 Introducción

La empresa multicines Ál-Ándalus ha llevado a cabo nuestra contratación con el fin de asentar las nuevas tecnologías del cine en Huelva. Consiguiendo así que los onubenses puedan disfrutar del cine convencional, 3D, 4D y con una resolución 4K.

Sin embargo, el software que la empresa ha estado usando está obsoleto para administrar su base de datos. Consideramos oportuna la implementación del software en el lenguaje Java solventando así la compatibilidad con la misma. Por lo tanto, el desarrollo debe comenzar desde cero dado que no se puede reutilizar código.

Tras una reunión con la empresa todos los requisitos se han validado a excepción de los relacionados con la venta de entradas por Internet. De todos ellos, los más prioritarios son los referentes a la venta en taquilla.

    • Descripción El sistema deberá comportarse tal como se describe en el siguiente caso de uso cuando el cliente desee reservar un número de entradas.
    Precondición Estar identificado en el sistema.
    Secuencia normal Paso Acción
    1 El actor Cliente (ACT-0001) indica al personal de taquillas: nombre, DNI, sala, sesión, número de entradas que desea reservar y fila que prefiere.
    2 El sistema propone un grupo de butacas contiguas de las disponibles en ese momento y permitir modificar la elección o rechazar la operación.
    3 El sistema marca las butacas reservadas y asigna automáticamente un identificador (código) para la reserva.
    4 El actor Personal de Taquillas (ACT-0002) comunica el identificador al cliente.
    Postcondición La reserva se ha hecho efectiva.
    Excepciones Paso Acción
    2 Si la reserva no se ha hecho efectiva, es decir, si no se han comprado las entradas, como muy tarde hasta dos días antes de la proyección, el sistema cancela la reserva, a continuación este caso de uso queda sin efecto

    UC-0004 Anular reserva
    Versión 1.0 ( 17/04/2018 )
    Autores ?
    Fuentes ?
    Dependencias
  • [OBJ-0002] Gestionar las actividades de la taquilla
    • Descripción El sistema deberá comportarse tal como se describe en el siguiente caso de uso cuando El usuario debe presentar el identificador de la reserva y su DNI. El cliente podrá realizar una anulación parcial o total. En la parcial, se deberá especificar el número de entradas a anular y seleccionar las butacas correspondientes, que pasarán a estar disponibles para nuevos consumidores. Por último, se mantendrá el registro de las reservas y se cambiarán los datos de las butacas de la forma apropiada.
    Precondición Se requerirá haber reservado una o más entradas previamente
    Secuencia normal Paso Acción
    1 El sistema comprobará al usuario y el identificador suministrado por el cliente, asegurandose que ambos son correctos
    2 El sistema mostrará las entradas reservadas por el cliente, permitiendole seleccionar aquellas butacas que desea cancelar. Si se desea cancelar todas las reservas se llamará al caso de uso [_]
    3 El sistema guardará toda la información correspondiente a la reserva anulada.
    Postcondición El sistema volverá a mostrar las butacas como disponibles
