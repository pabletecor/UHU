1 Introducción

La empresa multicines Ál-Ándalus ha llevado a cabo nuestra contratación con el fin de asentar las nuevas tecnologías del cine en Huelva. Consiguiendo así que los onubenses puedan disfrutar del cine convencional, 3D, 4D y con una resolución 4K.

Sin embargo, el software que la empresa ha estado usando está obsoleto para administrar su base de datos. Consideramos oportuna la implementación del software en el lenguaje Java solventando así la compatibilidad con la misma. Por lo tanto, el desarrollo debe comenzar desde cero dado que no se puede reutilizar código.

Tras una reunión con la empresa todos los requisitos se han validado a excepción de los relacionados con la venta de entradas por Internet. De todos ellos, los más prioritarios son los referentes a la venta en taquilla.

    UC-0001 Reservar entradas
    Versión 1.0 ( 10/04/2018 )
    Autores
    • Raúl Castilla Bravo
    Fuentes ?
    Dependencias
  • [IRQ-0006] Informacion de la Proyección
  • [IRQ-0003] Información de reserva
  • [OBJ-0003] Gestionar la venta de entradas por Internet
  • [OBJ-0002] Gestionar las actividades de la taquilla
    • Descripción El sistema deberá comportarse tal como se describe en el siguiente caso de uso cuando el cliente desee reservar una entrada.
    Precondición El Personal de Taquilla debe estar identificado en el sistema.
    Secuencia normal Paso Acción
    1 El sistema pide: nombre, DNI, sala y sesión.
    2 El actor Personal de Taquillas (ACT-0002) introduce los datos en el sistema
    3 El sistema propone un grupo de butacas contiguas de las disponibles en ese momento.
    4 El actor Personal de Taquillas (ACT-0002) selecciona las butacas a reservar de entre todas las disponibles
    5 El sistema marca la butaca como reservada y asigna automáticamente un identificador para la reserva.
    6 El actor Personal de Taquillas (ACT-0002) comunica el identificador al cliente.
    Postcondición La butaca seleccionada pasa a estar reservada.

  • [IRQ-0003] Información de reserva
    • Descripción El sistema deberá comportarse tal como se describe en el siguiente caso de uso cuando el Personal de Taquilla seleccione la opcion de Recogida de Entradas Previamente Reservadas
    Precondición El Personal de Taquilla deberá identificarse con su login en el sistema y un cliente físico ha de pasarse a recoger sus entradas previamente reservadas.
    Secuencia normal Paso Acción
    1 El actor Personal de Taquillas (ACT-0002) solicitará al cliente su DNI.
    2 El sistema buscará la reserva que posee dicho Cliente en base al DNI que este le ha proporcionado.
    3 El sistema imprimirá las entradas asociadas a la reserva del Cliente.
    4 El sistema marcará la/s butaca/s correspodiente a la/las entradas como ocupadas.
    5 El actor Personal de Taquillas (ACT-0002) realizara el cobro al Cliente.
    6 El actor Personal de Taquillas (ACT-0002) entregará las entradas imprimidas al Cliente
    Postcondición Las butacas quedarán marcadas como vendidas.

    • Descripción El sistema deberá comportarse tal como se describe en el siguiente caso de uso cuando el personal de taquillas haya seleccionado la opción vender entradas.
    Precondición El personal de la taquilla debe haberse identificado.
    Secuencia normal Paso Acción
    1 El sistema solicita un asiento.
    2 El actor Personal de Taquillas (ACT-0002) selecciona el asiento deseado por el cliente.
    3 El sistema guarda los cambios (venta efectuada y asientos seleccionados) en la Base de Datos.
    4 El sistema muestra un mensaje donde se hace saber que la asignación se ha llevado a cabo.
    Postcondición El sistema imprimirá las entradas.Además, las butacas se marcarán como vendidas.
    Excepciones Paso Acción
    2 Si la butaca seleccionada ya habia sido elegida por otro cliente con anterioridad, el sistema mostrará un mensaje donde se indique el error producido, a continuación este caso de uso queda sin efecto

    5 El actor Personal de Taquillas (ACT-0002) confirma la anulación de la reserva
    6 El sistema eliminará la reserva y cambiará el estado de la butaca a disponible
    Postcondición El sistema volverá a mostrar la butaca como disponible
